# 安全说明

请不要在公开 Issue 中粘贴飞书 token、Cookie、OAuth 信息、文档快照或私人课件。

仓库上线后，安全问题优先通过 GitHub Security Advisory 私下报告。报告中说明受影响版本、复现条件、可能影响和已经尝试的缓解方式。可以使用经过脱敏的最小示例，不要上传真实用户数据。

以下情况按安全问题处理：

- 安装包意外包含凭证、文档或本机路径；
- 路径检查可以越过当前工作目录；
- Update 模式能够绕过 revision 或 block 检查；
- 用户身份失败后自动改用其他身份；
- 命令参数可以造成命令注入；
- 画板更新意外覆盖文档中无关内容。

普通安装问题、OCR 质量和排版建议可以使用公开 Issue。
